### 版本8

- 新增能力级:rap,是否支持自研云rap协议，0-不支持，1-支持
- 新增能力级:dst,是否支持夏令时设置,0-不支持,1-支持
- 新增能力级:art1,是否支持报警声音类型设置,该能力是art的改进版,采用bitmap形式,每一个bit位表示是否支持该报警声音类型,0-不支持,bit0 - 您已进入监控区域请立即离开, bit1 - 禁止停车, bit2 - 你好欢迎光临
- 新增能力级:stf,屏幕上的时间显示是否支持小时设置 0 - 不支持，1 - 支持 (ScreenTimeFormat)
- 新增能力级:ups,0 - 不支持,1 - 内置, 2 - 外置
- 新增能力级:nup,是否支持物模型服务:notifyUploadProp, 0 - 不支持， 1 - 支持
- 新增能力级:dic,是否支持物模型:DistortionCorrection, 0 - 不支持， 1 - 支持
- 新增能力级:alm2,是否支持直播时app发起报警铃声,物模型类型为服务,0-不支持,1-支持
- 新增能力级:vqc,卡录像查询是否支持压缩, 0 - 不支持, 1 - 支持
- 新增能力级:mcl ,是否支持多目联动设置, 0 - 不支持, 1 - 支持
- 新增能力级:rec ,是否支持录像搜索IsReady、StorageStatus字段扩展, 0 - 不支持, 1 - 支持
- 新增能力级:wxc ,微信对讲方案，0-云云对接，1-纯云方案（新方案，设备不需要集成wx_voip）
- 新增能力级:vdt ,视频解码能力，采用bitmap形式，每个bit位表示设备支持解码类型: 0-不支持, bit0-H264, bit1-H265
- 新增能力级:vdf ,视频解码帧率上限，fps
- 新增能力级:drw ,屏幕分辨率，宽，pixel
- 新增能力级:drh ,屏幕分辨率，高，pixel
- 新增能力级:bec ,是否支持录像根据startTime和eventType播放及其播放方式是否带duration, 0 - 不支持, 1 - 支持
- 新增能力级:rbl ,是否支持录像设置,采用bitmap形式,bit位为1表示支持改码流,0表示不支持, bit0 - 存储优先, bit1 - 均衡, bit2 - 画质优先
- 新增能力级:osts ,是否支持使用STS Token访问OSS, 0 - 不支持,1 - 支持
- 新增能力级:eis ,防抖开关设置能力级, 0 - 不支持,1 - 支持
- 新增能力级:lf , 直播帧率能力级,采用bitmap形式，0表示不支持，bit0-15fps bit1-25fps bit2-30fps
- 新增能力级:ote ,时间戳水印开关, 0 - 不支持,1 - 支持
- 新增能力级:iis ,能力集 iis    图像调节-ISO{ImageISO;ISOAutoMax;ISOManuMax} 是否支持 0-不支持 else-表示自动和手动的最大范围值
- 新增能力级:icp ,图像调节-色温 设置能力级, 采用bitmap形式，0表示不支持，bit0-自动 bit1-2800 ;bit2-3800 ;bit3-4000 ;bit4-5000 ;bit5-6500 ;bit6-7500
- 新增能力级:cs ,图像调节-快门, 采用bitmap形式 0表示不支持，bit0-自动;bit1-1/50;bit2-1/200;bit3-1/500;bit4-1/2000
- 新增能力级:vse ,录像设置的入口是否显示，仅限于跟设备端存储相关的信息, 0 - 不支持,1 - 支持,是否最终显示，APP根据这个能力和其他{云存是否上报等}综合决定
- 新增能力级:zoom ,对焦能力集, 采用bitmap形式，每个bit位为1表示支持,0表示不支持,bit0=是否支持对焦能力总开关,bit1=支持对焦放大,bit2=支持对焦缩小
- 新增能力级:zoom_param_cap ,对焦参数能力集(变焦范围、步长、切sensor倍率), 详见下面参数描述
- 新增能力集:wpc ,自定义壁纸能力集 0-不支持 1-支持，不存在该字段时认为不支持
- 新增能力集:flk ,flicker 50/60 HZ的设置 0-不支持 1-支持，不存在该字段时认为不支持
- 新增能力集:sei ,是否支持录像SEI设置 0-不支持 1-支持，不存在该字段时认为不支持
- 新增能力集:wlbn ,是否支持暖光灯亮度调节能力 0-不支持 1-支持，不存在该字段时认为不支持
- 新增能力集:lbat 是否支持自动补光灯时间段  0-不支持 1-支持
- 新增能力集:eds1 是否支持自动补光灵敏度设置  0-不支持 1-支持  注意，和eds互斥，代码实现，不写入json文件,但是eds必须存在
- 新增能力集:cupc 是否号码牌抓拍上传开关和间隔配置，0-不支持 1-支持
- 新增能力集:lwm 开启直播必须的工作模式：bitmap，参考wm
- 新增能力集:cle 是否支持充电灯语开关  0-不支持 1-支持
- 新增能力集:tr 是否支持定时重启开关  0-不支持 1-支持
- 新增能力集:fvf 是否支持强制展示特写开关, 多通道设备按通道打能力则APP按通道展示特写画面（PT4设备，强制特写以单独窗口显示，比如PT4双目设备，全景强制特写打开，就会有第三个显示窗口显示特显）  0-不支持 1-支持
- 新增能力集:sns1 超级夜景V2的能力级,0 - 不支持,1 - 支持  注意，和sns互斥，只能有其一
- 新增能力集:amt 算法画框追踪 按位解析，每个bit位对应相应的算法是否支持画框, 0 - 不支持画框, 1 - 支持画框  对应物模型：AlgorithmMarkTrack

- 新增能力集: spkd 是否支持音频输出外设   bitmap  bit0-内置喇叭  bit1-3.5mm音频孔
- 新增能力集: anr 是否支持音频降噪  bitmap  bit0-关闭  bit1-人声 bit2-音乐 bit3-自然
- 新增能力集: lru 是否支持RTMP推流地址  0-不支持 1-支持
- 新增能力集: lapfps 运动相机延时摄影文件帧率
- 新增能力集：parts 是否支持配件，按bit位：0 不支持，1: 支持  --- (举例：V1设备知否支持拓展屏) bit0 - 拓展屏

- 新增: com (通用能力集---每个能力集占4位) 
      - bit4 ~ bit7, bit4:CBR,bit5:VBR,bit6:AVBR,bit7预留，全0表示不支持
      - bit8 ~ bit11, 设备拉流上行视频旋转角度，0: 0度，1：90度，2：180度，3：270度，M1默认1，其他默认0。备注：如果老版本没有com能力集，默认就是0
      - bit12 ~ bit15 支持紧急联系人的个数 
- 新增: com8 (通用能力集---每个能力集只占1位)  
        - bit2:  是否支持主码流设置分辨率 0-不支持,1-支持
        - bit3:  是否支持主码流设置码率 0-不支持,1-支持
        - bit4:  是否支持4G下文件下载 0-不支持,1-支持 （相册图片+mp4视频）
- 新增：gri （gps定位查询间隔，app根据这个时间定时来查询）
